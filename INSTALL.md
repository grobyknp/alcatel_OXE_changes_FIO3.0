Инструкция: подготовка окружения и запуск change_FIO2.1.py
Шаг 0. Установка Python (если ещё не установлен)
Откройте командную строку от имени администратора.

Введите python --version.
Самопроверка: Если вы видите версию (например, Python 3.12.2) → Python уже есть, переходите к шагу 1.
Если ошибка («не распознана»), скачайте Python с официального сайта. При установке обязательно отметьте галочку «Add Python to PATH». После установки перезапустите командную строку от администратора и повторите проверку.

Шаг 1. Создание виртуального окружения
Перейдите в папку, где лежит ваш скрипт change_FIO2.1.py:

cmd
cd C:\путь\к\вашей\папке
Создайте виртуальное окружение (назовём его venv):

cmd
python -m venv venv
Самопроверка: После выполнения появилась папка venv в текущей директории. Внутри неё есть подпапки Scripts и Lib.

Шаг 2. Активация виртуального окружения
Активируйте окружение (Windows):

cmd
venv\Scripts\activate
Самопроверка: В начале командной строки появился текст (venv).
Дополнительная проверка: введите where python — первым в списке должен быть путь C:\...\venv\Scripts\python.exe (внутри вашей папки).

Шаг 3. Установка дополнительных библиотек (если бы они были)
В вашем случае скрипт использует только стандартную библиотеку, поэтому ничего устанавливать не нужно.
Если бы требовались сторонние пакеты, команда была бы pip install <имя_пакета>, а проверка — pip list.

Шаг 4. Сохранение списка зависимостей в requirements.txt
Выполните:

cmd
pip freeze > requirements.txt
Самопроверка: Файл requirements.txt появился в папке. Откройте его блокнотом:

Он может быть пустым (если нет сторонних пакетов) — это абсолютно нормально.

Если внутри есть строчки (например, pip==23.x.x), они относятся к служебным инструментам самого pip и не обязательны для работы скрипта.

Пустой файл означает, что ваш код не требует установки каких-либо пакетов через pip — всё необходимое уже есть в стандартном Python.

Шаг 5. Упаковка requirements.txt в архив (по требованию)
Например, создайте ZIP-архив через командную строку (если у вас установлен tar или используйте любой архиватор вручную):

cmd
tar -a -cf requirements_archive.zip requirements.txt
Самопроверка: Появился файл requirements_archive.zip, внутри него лежит requirements.txt.
Альтернатива: просто скопируйте requirements.txt в любой архив через проводник.

Шаг 6. Запуск вашего скрипта в виртуальном окружении
Убедитесь, что в командной строке всё ещё видно (venv). Затем выполните:

cmd
python change_FIO2.1.py
Самопроверка: Скрипт запустился без ошибок импорта (например, ModuleNotFoundError). Если он что-то выводит или выполняет свою задачу — значит, окружение работает корректно.

Итоговая самопроверка всего процесса
Python установлен и виден в командной строке.

Папка venv создана.

Команда activate привела к появлению (venv).

pip freeze > requirements.txt создал файл (пустой — допустимо).

Архив с requirements.txt создан (если требовалось).

Скрипт change_FIO2.1.py успешно запустился из-под (venv).
